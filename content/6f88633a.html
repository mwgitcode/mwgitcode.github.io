一、概述

关于 Flutter SDK的下载及安装、环境配置等这里不做说明，有需要的朋友可以在网上找或直接在 Flutter 的官网查看即可，说的都很详细明白。从本章起，将介绍 Flutter 的基础组件知识，对每个部件做详细的使用说明，从 main 入口函数开始讲解，在学习的过程中理解如何使用。以后代码所使用的开发工具为 Android Studio ，模拟器使用的是 Iphone 模拟器，操作系统为 Mac 。

3. 工程目录结构

新建的工程目录如下图：

其中，开发 Flutter 工程代码主要关心的文件为：lib文件夹和 pubspec.yaml 配置文件。

lib文件夹：工程代码文件夹，里面有一个 main.dart 文件，为工程的入口文件，默认创建的工程包含了默认的demo代码。我们自己创建的代码文件也存放在lib文件夹中。

pubspec.yaml：工程的配置文件，用来管理第三方依赖和静态资源配置。

android 文件夹：Flutter 工程在 Android 平台运行的相关代码。

ios 文件夹：Flutter 工程在 ios 平台运行的相关代码。

build 文件夹：编译后的 Android 或 iOS 包。

test 文件夹：存放测试代码。

以点 . 开头的文件或文件夹为隐藏文件夹，以下是隐藏文件的说明：

.idea 文件夹：存储了 Android Studio 的配置信息，因为 Android Studio 是基于 Intelli IDEA IDEA 开发，因此有名为这个的文件夹，可以忽略此文件夹。

.gitignore：里面说明了Git版本控制要忽略的文件。

.metadata：Android Studio 用来跟踪当前项目的数据，可以忽略。

.packages：Flutter 带有包管理器，用于管理项目中的依赖项，此文件用于跟踪项目中的依赖关系，可以忽略。

pubspec.lock：Pub内部使用的文件，与pubspec.yaml 相同，不能编辑此文件，忽略即可。

study_flutter_demo.iml：Android Studio 的项目配置文件，可以忽略。

README.md：自述文件。

有兴趣的可以参考官方文档：https://dart.dev/guides/libraries/private-files 。
