[iOS.Metal]Metal基础知识(一)

发表于 2020-10-28 | 分类于 移动开发 > iOS开发

一、概述

Metal 是苹果 (Apple) 在2014 年推出的底层 GPU 编程框架。与 OpenGL ES 类似，均是用于与 3D 图形硬件交互的低级 API 。区别在于 Metal 不是跨平台的，而 OpenGL 是跨平台的，但是 Metal 相对 OpenGL ，在 Apple 平台上具有更高的速度和更低的开销，可以最大限度的利用 GPU 的能力。

GPU ( Graphics Processing Unit )，是专门处理图形图像的处理器，可以非常快速和高效地进行并行浮点运算，这些运算如果运行在 CPU ( Central Processing Unit ) 上的代价是非常昂贵的。

Metal 是支持 GPU 加速的高级3D图形渲染和数据并行计算的框架。它为图形和计算命令的组织、处理、提交以及管理这些命令的相关数据和资源提供了一个现代化的、精简的 API 。Metal 的主要目标是最大程度地减少执行 GPU 工作负载所导致的 CPU 开销。

很多高级 Apple 框架都建立在 Metal 之上，以利用其性能，包括 Core Image ，SpriteKit ，SceneKit 。可以直接使用这些高级框架来避免接触 GPU 编程的细节，但是编写自定义的 Metal 代码可以获得更高的性能。

Metal 框架提供以下功能：

  • 低开销的接口：Metal 皆在消除隐藏的性能瓶颈，比如隐式状态验证。可以控制 GPU 的异步行为，以实现用于并行创建和提交命令缓冲区的高效多线程。
  • 内存和资源管理：Metal 框架描述了代表 GPU 内存分配的缓冲区和纹理对象。纹理对象具有特定的像素格式，可以用于纹理图像和附件。
  • 对图形和计算操作的集成支持：Metal 对图形和计算操作使用相同的数据结构和资源（例如缓冲区，纹理和命令队列）。此外，Metal 着色语言支持图形和计算功能。 Metal框架使资源可以在运行时界面，图形着色器和计算功能之间共享。
  • 预编译的着色器：Metal 着色器可以在构建时与应用程序代码一起编译，然后在运行时加载。此工作流提供了更好的代码生成以及更容易的着色器代码调试。(金属还支持着色器代码的运行时编译)

二、基本实现

要完成一个基本的 Metal 设置，大概需要如下几个步骤：

  1. 获取 GPU 设备
  2. 创建渲染图层
  3. 设置顶点缓冲区
  4. 设置顶点着色器
  5. 设置片段着色器
  6. 设置渲染通道(管线)
  7. 提交到命令队列

作为开篇的基础文章，本文首先从最基本的设置开始，完成在 Metal 中清屏为纯色的实现。上述过程在本文并不会一一实现，只做最基础的说明，后续文章会陆续详细讲解。

1. 获取 GPU 设备

MTLDevice (定义在 Metal 框架) ：在 Metal 中，GPU 由一个符合 MTLDevice 的协议对象表示。MTLDevice 协议定义了表示单个 GPU 的接口，负责创建和管理用于处理数据并将数据呈现到屏幕的各种持久性的和瞬时的对象。也就是说，MTLDevice 对象表示可以执行命令的 GPU 。可以通过 MTLDevice 协议查询设备属性，创建其他特定于设备的对象，例如缓冲区和纹理，以及对提交给 GPU 执行的渲染和计算命令进行编码和队列操作。细化来说， MTLDevice 可以创建命令队列，队列中包含命令缓冲区，命令缓冲区中又包含编码器，编码器用于设置附加的 GPU 命令。通过使用 MTLDevice 可以发出所有 Metal 命令。

3. 获取可绘制对象

CAMetalDrawable ：Core Animation 为可显示资源对象定义了 CAMetalDrawable 协议 (不要自己实现此协议)。 实现此协议的对象属于 CAMetalLayer 对象。CAMetalDrawable 协议扩展了MTLDrawable 并提供了一个符合 MTLTexture 协议的对象，因此它可以用作呈现命令的目标。要渲染到CAMetalLayer 对象中，应该为每个渲染过程传递一个新的 CAMetalDrawable 对象，获得它提供的MTLTexture 对象，并使用该纹理创建颜色附件。与颜色附件不同，深度或模板附件的创建和销毁成本很高。如果需要深度或模板附件，请创建一次，然后在每次渲染框架时重复使用它们。

在 CAMetalLayer 内部会创建一个 Metal 的可绘制对象( CAMetalDrawable )池，在任何给定的时间，这些可绘制对象池中的一个( CAMetalDrawable 对象 )都包含该图层的内容。要在层中显示 Metal 所呈现的内容，必须从 CAMetalLayer 的可绘制对象池中获得一个可绘制的资源( CAMetalDrawable 对象 )，然后通过将该资源附加到 MTLRenderPassDescriptor 对象来呈现该资源中的纹理。在每次渲染新的一帧时调用其 nextDrawable 方法。如下：

4. 创建渲染过程描述

MTLRenderPassDescriptor ：渲染过程描述符， class 类型，可以看作是一组附件的集合，用于配制渲染过程要达到的效果。也可以为渲染过程生成的可见性信息设置目标缓冲区。

在 GPU 编程中，基本上是将要发送到 GPU 的数据与任何解释该数据的设置进行捆绑发送，而这些解释数据的设置就被封装在 MTLRenderPassDescriptor 中。该设置可以有一个或多个，但是大多数情况下，该设置被限制为仅有一个。

MTLRenderPassDescriptor 的属性可以设置颜色 ( Color )、深度 ( Depth )和模版 ( Stencil )像素数据。

5. 创建命令队列

MTLCommandQueue (定义在 Metal 框架) ：用于组织提供给 GPU 执行的命令缓冲区的队列，是一个协议，不要主动定义实现此协议。MTLCommandQueue 对象将命令缓冲区的有序列表排队等待 MTLDevice 执行。命令队列是线程安全的，并允许同时编码多个未完成的命令缓冲区。

要创建命令队列，调用 MTLDevice 对象的 makeCommandQueue() 或makeCommandQueue(maxCommandBufferCount:) 方法。由 makeCommandQueue(maxCommandBufferCount:) 方法返回的队列限制未完成命令缓冲区的数量。通常，您会在应用程序启动时创建一个或多个命令队列，然后在应用程序的整个生命周期中保留这些队列。

6. 获取命令缓冲区

MTLCommandBuffer ：是一个容器，用于存储编码的命令以供 GPU 执行，是一个协议，不要自己实现此协议。要呈现图像或执行计算操作，应通过命令队列 ( MTLCommandQueue ) 创建一个或多个命令缓冲区对象，然后将命令编码到这些对象中，并将它们提交到队列中。有两种方法可以创建 MTLCommandBuffer 对象，通过MTLCommandQueue 的 makeCommandBuffer() 和 makeCommandBufferWithUnretainedReferences() 。在大多数情况下，应使用 makeCommandBuffer() 方法，因为它创建的命令缓冲区，保存了 Metal 在完成命令缓冲区中编码的命令所需的任何对象的强引用。在极少数的情况下，可以使用 makeCommandBufferWithUnretainedReferences() 方法创建一个命令缓冲区，该缓冲区不保留对这些对象的强引用。在这种情况下，您要负责使这些对象保持活动状态，直到命令缓冲区中编码的命令执行完毕。

在使用相应的 MTLCommandQueue 创建命令缓冲区对象后，只能将该命令缓冲区提交到创建它的 MTLCommandQueue 上执行。所有发送到单个命令队列的命令缓冲区都保证按照命令缓冲区入队的顺序执行。

在创建完命令缓冲区后，需要创建命令编码器对象 (MTLRenderCommandEncoder ) ，来对缓冲区对象进行填充。MTLCommandBuffer 协议具有创建特定类型的命令编码器的方法，可以创建如下四种类型的编码器：MTLRenderCommandEncoder ， MTLComputeCommandEncoder ，MTLBlitCommandEncoder 和 MTLParallelRenderCommandEncoder 。在任何给定时间，对于特定的命令缓冲区，只有一个编码器可以处于活动状态。可以创建多个命令编码器来向缓冲区添加任务，但是在创建另一个编码器之前，需要结束当前编码器的编码过程。也就是说，创建一个编码器，使用它向缓冲区添加命令，然后结束编码过程。在完成一个编码器之后，才可以创建另一个编码器，并继续向相同的缓冲区添加命令。当准备好执行这组已编码的命令时，可以调用命令缓冲区的 commit() 方法来调度缓冲区以便执行。

在调用 commit() 方法之前，需要调用 present(drawable:) 方法， present(drawable:) 方法用来注册一个可尽快绘制的对象，是一个便利方法。该方法会在命令队列调用命令缓冲区执行完毕之后调用 MTLDrawable 的 present() 方法。 MTLDrawable 的 present() 方法会跟踪命令缓冲区中的命令或可绘制对象。当缓冲区中的命令或可绘制对象请求完成时尽快显示其内容。
